# AI提示词边界设计

很多 AI 命理系统的问题，并不只出在模型本身，也出在提示词设计太宽。提示词如果只强调“回答完整、尽量帮助用户、保持专业”，模型就会天然往更满、更确定、更像专家断语的方向走，这恰恰是高风险来源之一。

提示词边界设计的核心，不是让模型说得更漂亮，而是让模型知道什么该答、什么不该答、什么时候该保守、什么时候该升级给人。

## 一、为什么提示词边界很重要

因为模型默认会尽量补全、尽量连贯、尽量像知道答案。如果系统没有明确约束，模型在信息不足或高风险场景里也会继续往下说。

## 二、提示词边界至少要控制什么

建议至少控制四件事：

1. 角色边界
2. 任务边界
3. 语言边界
4. 升级边界

## 三、角色边界怎么设计

要先告诉模型，它是：

1. 知识整理与解释工具
2. 结构化辅助判断工具
3. 不是医疗、法律、财务等现实专业系统替代品

这一步能直接降低模型把自己包装成绝对权威的倾向。

## 四、任务边界怎么设计

需要明确区分不同任务：

1. 规则计算题
2. 术语解释题
3. 结构分析题
4. 高风险现实问题

不同任务的回答深度、结论强度和边界提示都不应一样。

## 五、语言边界怎么设计

提示词里应明确约束：

1. 信息不足时必须显式说明
2. 避免使用“必然”“一定”“绝对”等强结论词
3. 有知识冲突时需要表达不确定性
4. 高风险场景中优先提示边界和现实专业支持

这部分直接关系到 [AI过度自信案例：什么时候模型最容易把不确定说成确定](ai-overconfidence-cases.md)。

## 六、升级边界怎么设计

成熟系统不应只靠一句“仅供参考”收尾，而是要在提示词中定义：

1. 哪些问题要降低结论强度
2. 哪些问题要要求补充信息
3. 哪些问题必须提示转向现实专业系统
4. 哪些问题需要人工复核或接管

## 七、提示词设计最常见的失败方式

### 1. 只强调“尽量有帮助”

结果模型把“有帮助”理解成“无论如何都给满答案”。

### 2. 边界规则只放在末尾免责声明

如果主体提示仍在鼓励强输出，末尾免责声明往往压不住整体倾向。

### 3. 没有按任务拆提示词

规则题、解释题和高风险题如果共用一套风格，很容易互相污染。

## 八、怎样判断提示词边界是否有效

可以看四点：

1. 信息不足时是否明显更克制
2. 高风险题是否更容易触发升级或转介
3. 回答语气是否降低了绝对化表达
4. 失败案例是否显著减少
